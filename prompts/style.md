(生成或修改文档时遵守的技术写作规范)

1. 语态/时态:主动语态、一般现在时。写 "Returns the URL",不写 "The URL will be returned"。
2. 人称:面向读者用第二人称(you / 调用方),避免 "we"。
3. 简洁:一句话只说一件事;删掉 simply / just / basically / note that 这类废词。
4. 一致性:同一概念全程用同一术语(别 link / URL / 短链 混用);大小写、代码标记风格统一。
5. 首次出现即定义:缩写或专有名词第一次出现时给出全称或一句解释。
6. 并列结构:列表项、参数说明用同样句式开头(要么都用动词,要么都用名词)。
7. 描述 API:先一句话说它做什么,再说参数、返回值、错误/边界情况。
8. 标记:代码、标识符、字段名用反引号包裹;示例尽量可运行、最小化。
9. 不臆造:只写代码确实支持的行为,不确定就不写。
